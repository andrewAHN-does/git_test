폰트

기본 - noto sans kr
영문 - Montserrat

HTML 가이드

  • 공통 클래스에는 'g_' 추가 eg) g_common
  • 공통 타이틀에는 'tit_' eg) tit_h2
  • 공통적인 레이아웃에는 section 클래스 추가
  • 일부 레이아웃이 달라져야 하는 페이지에는 section 클래스 뒤 해당 페이지명을 붙임. eg) class="section about1"

CSS 가이드

  • 선택자는 최대 6개 이하
  • 기본적으로 아이디나 클래스 단위로 선택, 특수한 경우가 아니면 태그와 선택자를 동시에 사용하지 말것. eg) .selector(O) / a.selector(X)
  • 공통부분 등 상단에 주석을 필수로 기입
  • !important는 최대한 지양한다.
  • css 네이밍의 단어와 단어 사이에는 ‘_’를 사용한다.
  • 활성화는 on 으로 통일한다.

JS 가이드

  • 네이밍의 첫 자는 소문자 단어와 단어 사이는 대소문자로 구분한다
  • 네이밍이 문자, 숫자 조합일 경우 문자를 앞에 사용한다 (문자+숫자)
  • !important는 최대한 지양한다.
  • UI 별로 주석을 필이 기입
  • 변수
    - 전역변수 : g_
    - 지역변수 : m_
    - jquery selecter : $
    - 배열 : arr
    - 버튼 : btn
    - boolean : is
    - for문 : i > j > k (다중 for문일 경우)
    - 인자값 : _
